Nested rules: @media rules style editing

This test case covers one part of issue Issue 1717 (CSS Panel does not have @media UI).

Steps to reproduce

  1. Open Firebug
  2. Switch to the CSS panel
  3. Select atMediaStyleEditing.html from the Location Menu
    ⇒ There must be one unnested #element rule, one nested inside an @media all and one inside an @media print rule.
  4. Click the value of the background-image property inside the #element rule nested inside the @media all rule
    ⇒ The inline editor should open.
  5. Enter -moz-linear-gradient(135deg, #FF8C78, #FFC8B4)
    ⇒ The <div> (#element) above should be red now.
  6. Click the disable icon besides the background-image property inside the #element rule nested inside the @media all rule
    ⇒ The property should be displayed disabled inside the panel.
    ⇒ The <div> (#element) above should be blue now.

Expected result

  • Style editing inside @media rules should work as expected.
