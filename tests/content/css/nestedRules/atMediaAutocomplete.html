Nested rules: @media rules auto-completion

This test case covers one part of issue Issue 1717 (CSS Panel does not have @media UI).

Steps to reproduce

  1. Open Firebug
  2. Switch to the CSS panel
  3. Select atMediaAutocomplete.html from the Location Menu
    ⇒ There must be one unnested #element rule, one nested inside an @media all and one inside an @media print rule.
  4. Click the media type of the @media print rule
    ⇒ The inline editor should open.
  5. Press Down
    ⇒ The value should be projection now and the inline editor should be surrounded by a green border to indicate that saving the value was successful. ⇒ The <div> (#element) above should still be green.
  6. Type s
    The value should be auto-completed to screen and the inline editor should be surrounded by a green border to indicate that saving the value was successful.
    ⇒ The <div> (#element) above should be yellow now.

Expected result

  • Auto-completion for media types of @media rules should work as expected.
