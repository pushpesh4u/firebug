Issue 3303: Up/Down key working incorrectly for fonts when editing CSS

Hello Firebug user!

Steps to reproduce

  1. Open Firebug
  2. Switch to the HTML panel and there to the Style side panel
  3. Inspect the paragraph above (#font)
  4. Click on the value of the font-family property inside the #font rule
  5. Set the text cursor between the A and the l of Arial
  6. Press Down until you reach a font with spaces in it
    The value should now be something like <font with spaces>,serif with the font with spaces being selected
  7. Press Down until you reach a font without spaces in it
    The value should now be something like <fontwithoutspaces>,serif with the font without spaces being selected
  8. Press Up until you reach the font with spaces again
    The value should now be <font with spaces>,serif again with the font with spaces being selected
  9. Press Up until you reach Arial again
    The value should now be Arial,serif again with Arial being selected

Expected result

  • Auto-completion should work correctly on space-separated font names when pressing Up/Down
