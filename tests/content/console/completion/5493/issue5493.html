Issue 5493: Case-insensitive auto-completion

Steps to reproduce

  • Open Firebug and enable the Console panel.
  • Verify the following completions by typing the prefix and pressing TAB:
    • "obj.A" ⇒ "obj.AbcD"
    • "obj.a" ⇒ "obj.aBcdE"
    • "obj.AB" ⇒ "obj.AB"
    • "obj.ab" ⇒ "obj.aBdeE"
    • "obj.abx<backspace>" ⇒ "obj.AbcD"
    • "obj.Abcd" ⇒ "obj.AbcD"
    • "document.gete" ⇒ "document.getElementById"
    • "decodeu" ⇒ "decodeURI"
    • "obje" ⇒ "obje"
    • "Obje" ⇒ "Object"
  • Verify that when typing ";obj.ab", the completion text starts with that, but the popup has the correct case (and excludes the semicolon, as per issue 5999).

Expected Results

  • Case-insensitive matches should appear as completions.
  • Typed capital letters should force case at those points.
  • Completions that match case should be preferred as defaults.
  • When typing another letter, the completion should remain unless another completion suddenly appears better at matching case.
  • For global completions, the first letter should not be matched case-insensitively.
